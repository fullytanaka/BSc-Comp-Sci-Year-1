Project: online-shop
Authors: Michael Kölling and David J. Barnes

This project is part of the material for the book

   Objects First with Java - A Practical Introduction using BlueJ
   Sixth edition
   David J. Barnes and Michael Kölling
   Pearson Education, 2016

This project implements a small part of an online sales system (such as Amazon.com).
The current project is concerned only with customer comments for sales items. It contains
code to create, store, show and manipulate customer comments.

NOTE: There are several serious errors to be found in these classes.

## SGT Tutorial Questions

1.  Which aspects of a class need to be tested and debugged?

What the unit should do - its contract
Postive and negative tests - true and false
Test boundaries - one, zero, full
Blackbox testing vs Whitebox testing - testing for features vs testing for logic

All methods of a class need to be tested.

2.  What are the different kinds of errors that can occur?  What is a bug?

Syntax error - Stuff highlighted by the IDE
Logical error - Stuff that don't work as intended

3.  What is the difference between testing and debugging?

Testing searches for the presence of errors.
Debugging searches for the source of errors.

4.  What kinds of testing techniques are there?

Unit testing
Test automation - White box/black box

5.  What kinds of debugging techniques are there?

Manual walkthroughs
Debuggers
Print statements
